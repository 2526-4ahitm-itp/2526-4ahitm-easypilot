Technologies Used

The iOS stack

🎨 SwiftUI

Entire declarative UI — TabView, custom glass-card design system, live-binding telemetry views. One shared @StateObject owns the connection.

🌐 Network.framework

NWListener for UDP beacon discovery and URLSessionWebSocketTask for the bidirectional data link.

🧊 SceneKit

Real-time 3D flight simulator. A .usdz drone model, FPV + chase cameras, animated propellers, physics on the render loop.

📱 CoreMotion + Combine

CMMotionManager reads gyro/accelerometer at 10 Hz; @Published + ObservableObject push state into the UI.

Code · Networking

Zero-config discovery + WebSocket

The app finds the drone with no manual IP entry by listening for its UDP beacon, then upgrades to a WebSocket.

// Listen for the ESP32's UDP beacon on port 4242
private func receiveBeacon(on conn: NWConnection) {
    conn.receiveMessage { [weak self] data, _, _, _ in
        if let data,
           let text = String(data: data, encoding: .utf8),
           text.hasPrefix("EASYPILOT:") {
            let ip = String(text.dropFirst("EASYPILOT:".count))
            connectWebSocket(to: ip)   // ws://<ip>:81
        }
    }
}
Code · Live Telemetry

Decoding 10 Hz telemetry into the UI

// Codable struct mirrors the ESP32 JSON exactly
struct DroneTelemetry: Codable {
    let roll, pitch, yaw: Float
    let m1, m2, m3, m4: Int?
    let voltage: Float?, armed: Bool?, mode: String?
}

private func decodeJSON(_ text: String) {
    let decoded = try JSONDecoder().decode(DroneTelemetry.self, from: data)
    DispatchQueue.main.async {
        self.telemetry   = decoded   // @Published → SwiftUI redraws
        self.isConnected = true
    }
}

Challenges

What was hard — and how we solved it

🔍 Finding the drone

The ESP32 gets a random DHCP IP. Solution: a UDP beacon broadcasts the IP every 5 s so the app auto-connects — no typing required.

🔌 Detecting a dropped link

WiFi can silently die. Solution: a 3 s timeout timer + 5 s WebSocket pings flip isConnected false the moment data stops.

🧵 Thread safety

SceneKit physics runs on the render thread, SwiftUI only on main. Solution: physics writes _-vars, synced to @Published at 20 Hz on the main thread.

🛡️ Safety

Accidental motor spin is dangerous. Solution: 1.5 s hold-to-arm, debounced safe-test, and a tilt-over-60° emergency stop.

Code · The threading challenge

Two threads, one drone

Physics on SceneKit's render thread must never touch SwiftUI directly — so we separate internal state from published state.

class DroneSimulator: ObservableObject {
    // Main thread only — drives SwiftUI
    @Published var simRoll = 0.0
    @Published var isCrashed = false

    // Render thread only — raw physics integration
    var _roll = 0.0
    var _vel: SIMD3<Float> = .zero

    func tick(dt: Double) { /* render thread: physics */ }
    func syncPublished() { /* main thread: _vars → @Published */ }
}
