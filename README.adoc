= EasyPilot — FPV Drone with Companion App & Web Simulator

image:https://img.shields.io/badge/ESP32--C3-Firmware-red?style=for-the-badge[ESP32]
image:https://img.shields.io/badge/iOS-SwiftUI-black?style=for-the-badge[iOS]
image:https://img.shields.io/badge/Quarkus-3.28-4695EB?style=for-the-badge[Quarkus]
image:https://img.shields.io/badge/React-19-61DAFB?style=for-the-badge[React]
image:https://img.shields.io/badge/three.js-3D-049EF4?style=for-the-badge[three.js]

EasyPilot is a drone telemetry and control system built as a school project at
HTL Leonding (4AHITM). A custom ESP32-C3 board talks to a flight controller over
MSP, broadcasts telemetry, and accepts flight commands. An iOS companion app
discovers the drone on the network, streams live telemetry, and can fly an
on-device physics simulator. A small Quarkus service, a Python relay, and a
React + three.js web viewer round out the stack.

== Subsystems

[cols="1,2,3",options="header"]
|===
| Component | Location | Role

| ESP32-C3 firmware
| `PlatformIO/Projects/EasyPilotIOS/`
| WebSocket server (port 81) + UDP discovery beacon (port 4242). Drives the
  flight controller over MSP; implements IDLE / BALANCE / MANUAL / SOUND modes.

| iOS app
| `EasyPilotIOS/`
| SwiftUI app. Discovers the ESP32 via its UDP beacon, exchanges telemetry and
  commands over WebSocket, and includes a fully on-device flight simulator
  (rate / balance / POSHOLD, altitude-hold).

| Companion backend
| `webinterface/backend/`
| Quarkus 3.28 service exposing a small HTTP surface (`GET /api/status`).

| Telemetry relay
| `webinterface/backend/src/main/python/`
| Python WebSocket → UDP relay (`relay_server.py`) and an MSP stabilization
  sender (`websocket_sender.py`).

| Web viewer
| `webinterface/frontend/`
| React 19 + three.js 3D model viewer; subscribes to the relay over WebSocket.
|===

== Communication

[source]
----
ESP32-C3 (WiFi)
  ├── UDP broadcast "EASYPILOT:<IP>" every 5 s on port 4242  →  iOS discovers the drone
  └── WebSocket server on port 81  ←→  iOS sends commands / receives telemetry JSON @ 10 Hz

Python relay (off-board path)
  └── WebSocket :8080  →  UDP broadcast 255.255.255.255:4242  (feeds the iOS app remotely)

Web viewer
  └── WebSocket ws://localhost:8765  ←  websocket_sender.py
----

== Build & Run

=== ESP32-C3 firmware (PlatformIO)

[source,bash]
----
cd PlatformIO/Projects/EasyPilotIOS
# create secrets.ini (gitignored) with wifi_ssid / wifi_pass; load_secrets.py
# generates include/secrets_auto.h at build time
pio run            # build
pio upload         # flash
----

=== iOS app (Xcode)

[source,bash]
----
./deploy.sh [DEVICE_UDID]    # build + install on a connected iPhone
----

=== Companion backend (Quarkus)

[source,bash]
----
cd webinterface/backend
./mvnw quarkus:dev    # dev mode on http://localhost:8080
./mvnw test           # run the test suite
----

=== Telemetry relay (Python)

[source,bash]
----
cd webinterface/backend/src/main/python
pip install websockets pyserial
python3 relay_server.py       # WebSocket → UDP relay
# LOG_LEVEL=DEBUG python3 relay_server.py   # verbose per-message logging
----

=== Web viewer (React + Vite)

[source,bash]
----
cd webinterface/frontend
npm install
npm run dev
----

== Secrets

WLAN and OTA credentials are never committed. Copy the provided templates and
fill in your own values (the real files are gitignored):

* PlatformIO firmware: create `secrets.ini` (consumed by `load_secrets.py`).
* Standalone sketches: copy `wifi_secrets.example.h` → `wifi_secrets.h`.
